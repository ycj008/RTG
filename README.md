# RTG 高精度自动定位系统工程实施手册

---

## 一、项目背景

针对传统定位方案在港口复杂环境下存在精度不足、成本高及适应性差等痛点，本项目引入高精度 GNSS 差分定位技术。通过"基站 + RTG 接收端"架构，为 RTG 大车/小车提供实时、高频、精准的定位数据，以有效支撑自动化调度、大车纠偏及作业安全防护需求。

---

## 二、核心需求

为保障 RTG 自动化作业的精准度与稳定性，系统需满足以下核心要求：

- **全方位定位输出**：实时输出 RTG 大车/小车的绝对位置、运行速度及相对偏移量。
- **高精度技术指标**：平面定位精度达到 ±30mm，高度定位精度达到 ±50mm，且数据刷新率不低于 10Hz。
- **灵活适配机制**：支持堆场坐标系现场标定及标定数据复用，确保定位数据与电子地图高度匹配，无缝适配多堆场作业切换。
- **极高系统可靠性**：具备故障容错与丢星补偿能力，确保系统实现 7×24 小时连续、稳定运行。

## 三、坐标系统与定位基准

### 3.1 双坐标系融合

系统必须实时处理两种坐标系之间的转换：

- **WGS84 经纬度系统**：接收机获取的原始定位数据 `(Lat, Lon, Alt)`。
- **堆场局部坐标系 (LYCS)**：每个堆场拥有独立的物理原点 `(0, 0, 0)`，下发给 PLC 的数据须转换为每个堆场内的坐标。

## 四、硬件架构与通信链路

### 4.1 接收机与天线配置（3机6天线）

- **大车 (Gantry)**：2 台双天线接收机，部署于端梁四个角落。
  - 解算 **L-Point**（左侧门腿基准点）的 3D 坐标。
  - 实时输出解算航向角 (Heading)、横滚 (Roll) 及俯仰 (Pitch)。
- **小车 (Trolley)**：1 台双天线接收机，部署于小车两端对称位置。
  - 实时解算小车相对于 RTG 坐标系的 *X* / *Y* / *Z* 位置。

### 4.2 通信拓扑

| 链路 | 协议 | 说明 |
|------|------|------|
| 内部通信 | UDP/IP（以太网） | 接收机与边缘计算终端之间 |
| 外部通信 | Socket TCP（小端模式） | 工控机与 PLC 之间，帧率 ≥ 10Hz |

## 五、数学解算模型：L-Point 刚体变换

系统不直接使用天线坐标，而是通过"中转站"逻辑将坐标还原到地面作业点位。

### 5.1 静态参数说明

在解算前，必须在系统内录入车辆的物理几何参数：

| 参数 | 含义 |
|------|------|
| `L_arm` | 杆臂向量：天线相位中心相对于地面左侧作业对齐点 (L-Point) 在 RTG 局部坐标系下的 3D 偏移矢量 `(dx, dy, dz)` |
| `H` | 标称高度：天线相位中心到地面的垂直设计距离 |
| `W_span` | 跨距：左右门腿物理中心线的水平距离 |

### 5.2 L-Point 解算公式

为消除因车体倾斜导致的定位偏差，采用三维刚体旋转投影模型计算 L-Point 的世界坐标 `P_final`：

```
P_final = P_gps − R · L_arm
```

**参数说明：**

| 符号 | 含义 |
|------|------|
| `P_gps` | GNSS 接收机输出的实时三维天线位置 `(Xg, Yg, Zg)` |
| `L_arm` | 预设的 3D 杆臂补偿矢量 |
| `R` | 姿态旋转矩阵，由欧拉角构建：`R = Rz(ψ) · Ry(θ) · Rx(φ)` |

### 5.3 姿态角定义

1. **横滚角 (Roll, φ)**：绕 RTG 前后行驶轴（X 轴）旋转，反映 RTG 因地面不平导致的**左右倾斜**。
2. **俯仰角 (Pitch, θ)**：绕 RTG 左右门腿横向轴（Y 轴）旋转，反映 RTG 因轨道起伏导致的**前后倾斜**。
3. **航向角 (Heading, ψ)**：绕垂直轴（Z 轴）旋转，定义 RTG 在堆场中的**行驶朝向**。

### 5.4 B 车 3 点校准逻辑

B 车进场后，前 3 个作业点位用于求解转换矩阵 `M`，从而实现对 A 车底图的继承：

- **点 1**：求解平移偏差 `(ΔX, ΔY)`。
- **点 2**：求解旋转偏差 `(Δθ)`。
- **点 3**：冗余校验与尺度纠偏，锁定最终转换参数。

**最终输出：**

```
P_final_output = M × P_final
```

## 六、核心功能与解算逻辑

### 6.1 大车定位输出

- **双中心输出**：分别反馈电气房侧中心和柴油机侧中心距离堆场原点的实际距离。
- **纠偏原始数据**：实时输出四个门腿中心点相对跑道中心线的偏移量。
- **速度反馈**：大车值增大方向速度为正，减小为负。

### 6.2 小车定位解算

- **相对位置输出**：输出小车两端 GNSS 天线在 RTG 坐标系下的实时 *X* / *Y* / *Z* 坐标。
- **实际行程**：计算小车距离轨道起始点的实际运行距离。
- **安装补偿**：程序支持对小车 2 根天线的安装位置进行数据补偿，消除因安装空间受限导致的结构性偏差。
- **速度定义**：小车沿轨道运行数值增大方向为正，反之为负。

## 七、工程实施流程

### 第一阶段：RTK 手持打点（物理底座）

1. **测绘对象**：堆场原点、所有贝位 (Bay) 中心线、磁钉。
2. **操作规范**：在 RTK 固定解状态下静止采集，记录 10 帧取均值，建立 `ground_truth.db` 数据库。

### 第二阶段：A 车跑车建图（动态残差修正）

1. **参数录入**：输入 A 车天线杆臂向量 `L_arm` 及标称高度 `H`。
2. **残差解算**：A 车在贝位停靠，系统对比"手持打点值"与"大车实时解算值"。
3. **补丁生成**：记录 Z 轴垂直偏差，用于补偿路面不平导致的高程波动。

### 第三阶段：B 车校准（参数继承）

B 车进场后，在任意前三个点位执行校准，生成转换矩阵 `M`，使 B 车直接调用 A 车生成的 `yard_map.json`。

## 八、软件部署指南（Ubuntu 21.04）

### 8.1 系统优化

- **实时内核补丁**：安装 `rt-preempt`，确保解算任务的实时性（响应时间 ≤ 100ms）。
- **时间同步**：使用 `chrony` 配合 GPS PPS 信号，确保系统时钟微秒级对齐。

### 8.2 模块化部署

| 模块 | 功能 |
|------|------|
| `driver_node` | 解析 NMEA 报文及接收机专有格式数据 |
| `fusion_node` | 运行卡尔曼滤波 (EKF)，融合 IMU 与 GNSS 信号 |
| `solver_node` | 执行姿态投影、L-Point 变换及 `M` 矩阵修正 |
| `bridge_node` | 负责与 PLC 通信，下发 JSON/UDP 指令报文 |

### 8.3 丢星补偿（IMU 融合）

- **惯导补位**：GNSS 信号短时间中断（< 10s）时，利用内置 IMU 进行位姿推算，偏差需 < 250mm。

## 九、常见工程误差消除

1. **动态高度修正**：利用车校准时的已知点真值，反推实际工作高度，消除轮胎气压、负载导致的投影偏移。
2. **姿态平滑**：采用卡尔曼滤波融合 IMU 数据，过滤机械震动引起的航向角和位置跳变。
3. **自动跨堆场切换**：系统识别 RTG 所在的堆场区域，自动加载对应的标定参数和底图。
